# 2、解决跨域问题

跨域问题一直是前后端交互必须面临的问题，因此我们第二步是配置跨域，这里我们采用 CORS

在 demo 目录下新建 CorsConfiguration 类

package com.example.demo.config;

import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.CorsRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

/**
 * 配置跨域信息
 *
 * @author Jack
 */
@Configuration
public class CorsConfiguration implements WebMvcConfigurer {
    /**
     * 这里主要为了解决跨域问题,所以重写addCorsMappings方法
     *
     * @param registry CorsRegistry
     */
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")
                .allowedOrigins("*") // 允许所有来源
                .allowedMethods("GET", "HEAD", "POST", "PUT", "DELETE", "OPTIONS")
                .allowedHeaders("*")
                .exposedHeaders("access-control-allow-headers",
                        "access-control-allow-methods",
                        "access-control-allow-origin",
                        "access-control-max-age",
                        "X-Frame-Options")
                .allowCredentials(false).maxAge(3600);
        WebMvcConfigurer.super.addCorsMappings(registry);
    }
}

# 3、用户登录

# 1、前置知识

用户登录一般会涉及到状态保持的问题，也就是用户关闭页面，下次打开浏览器登录状态还在，不需要再重新登录，传统模式下一般可以用cookie 或者 session 保存用户的登录状态，但是现在我们用 jwt (JSON Web Token)进行状态保持，那我们来看看为什么不用 session

session 的弊端：

1、服务器压力增大 通常 session 是存储在内存中的，每个用户通过认证之后都会将 session 数据保存在服务器的内存中，而当用户量增大时，服务器的压力增大。

2、CSRF跨站伪造请求攻击 session 是基于 cookie 进行用户识别的, cookie 如果被截获，用户就会很容易受到跨站请求伪造的攻击。

3、扩展性不强 如果将来搭建了多个服务器，虽然每个服务器都执行的是同样的业务逻辑，但是 session 数据是保存在内存中的（不是共享的），用户第一次访问的是服务器1，当用户再次请求时可能访问的是另外一台服务器2，服务器2获取不到 session 信息，就判定用户没有登陆过。

Token 实现原理：

Token 是在服务端产生的。如果前端使用用户名/密码向服务端请求认证，服务端认证成功，那么在服务端会返回 Token 给前端。前端可以在每次请求的时候带上 Token 证明自己的合法地位。如果这个 Token 在服务端持久化（比如存入数据库），那它就是一个永久的身份令牌。 token 有很多种实现方式，这里以 jwt 为例子： JWT ： 是一种基于 JSON ，用于在网络上声明某种主张的令牌（token）

jwt 通常由三部分组成：头部信息( header ) 、消息体积和签名

  • 头部信息：指定了该 JWT 使用的签名算法

  • 消息体：包含 JWT中储存的信息

  • 签名：通过密钥跟算法生成签名

token 与 Session 的不同点：

token 与 session 的不同点主要在：

  1. 认证成功后，会对当前用户数据进行加密，生成一个加密字符串 token ，返还给客户端（服务器端并不进行保存）

  2. 浏览器会将接收到的 token 值存储在 Local Storage 中，（通过js代码写入 Local Storage ，通过 js 获取，并不会像 cookie 一样自动携带）

  3. 再次访问时服务器端对 token 值的处理：服务器对浏览器传来的 token 值进行解密，解密完成后进行用户数据的查询，如果查询成功，则通过认证，实现状态保持，所以，即时有了多台服务器，服务器也只是做了 token 的解密和用户数据的查询，它不需要在服务端去保留用户的认证信息或者会话信息，这就意味着基于 token 认证机制的应用不需要去考虑用户在哪一台服务器登录了，这就为应用的扩展提供了便利，解决了 session 扩展性的弊端。

# 2、JWT由三部分构成

第一部分我们称它为头部（header)，第二部分我们称其为载荷（payload, 类似于飞机上承载的物品)，第三部分是签证（signature).

header

jwt的头部承载两部分信息：

  • 声明类型，这里是jwt
  • 声明加密的算法 通常直接使用 HMAC SHA256

完整的头部就像下面这样的 JSON：

{  "typ": "JWT",  "alg": "HS256"}
1

然后将头部进行base64加密（该加密是可以对称解密的),构成了第一部分

eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9
1

playload

载荷就是存放有效信息的地方。这个名字像是特指飞机上承载的货品，这些有效信息包含三个部分

  • 标准中注册的声明
  • 公共的声明
  • 私有的声明

标准中注册的声明 (建议但不强制使用) ：

  • iss: jwt签发者
  • sub: jwt所面向的用户
  • aud: 接收jwt的一方
  • exp: jwt的过期时间，这个过期时间必须要大于签发时间
  • nbf: 定义在什么时间之前，该jwt都是不可用的.
  • iat: jwt的签发时间
  • jti: jwt的唯一身份标识，主要用来作为一次性token,从而回避重放攻击。

公共的声明 ： 公共的声明可以添加任何的信息，一般添加用户的相关信息或其他业务需要的必要信息。但不建议添加敏感信息，因为该部分在客户端可解密

私有的声明 ：

私有声明是提供者和消费者所共同定义的声明，一般不建议存放敏感信息，因为base64是对称解密的，意味着该部分信息可以归类为明文信息

定义一个payload:

{  "sub": "1234567890",  "name": "John Doe",  "admin": true}
1

然后将其进行 base64 加密，得到 Jwt 的第二部分

eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiYWRtaW4iOnRydWV9
1

signature

jwt的第三部分是一个签证信息，这个签证信息由三部分组成：

  • header(base64后的)
  • payload (base64后的)
  • secret

这个部分需要 base64 加密后的 header 和 base64 加密后的 payload 使用.连接组成的字符串(头部在前)，然后通过 header 中声明的加密方式进行加盐 secret 组合加密，然后就构成了 jwt 的第三部分。

UQmqAUhUrpDVV2ST7mZKyLTomVfg7sYkEjmdDI5XF8Q
1

密钥secret是保存在服务端的，服务端会根据这个密钥进行生成token和验证，所以需要保护好。

UserApi：

生成token字符串并设置有效期，为什么要设置有效期，因为为了安全，token被窃取也有安全隐患问题，所以为了安全一般会设置有效期，根据业务调整，这里设置 3600 * 24 * 7也就是7天的有效期。然后秘钥的话采用用户密码作为秘钥，这样能最大限度保证安全，因为即使秘钥泄露了，也不会导致所有token能被解密，因为每个用户的密码大多都不一样，所以只有被同样密码加密的token能被解密，如果使用统一秘密则都能被解密，十分危险。

/**
 * 签发token，并且设置有效期
 *
 * @param user 用户信息
 * @return token
 */
public String generateToken(User user) {
    Calendar instance = Calendar.getInstance();
    instance.add(Calendar.SECOND, 3600 * 24 * 7);
    return JWT.create()
            .withAudience(user.getUsername())
            .withExpiresAt(instance.getTime())
            .sign(Algorithm.HMAC256(user.getPassword())); // 使用用户密码作为私钥
}

# 3、获取用户信息

# 1、鉴权流程

有的方法是需要用户登录的后才能访问的，比如获取用户信息。那么，怎么判断用户的操作是否登录了呢？这时候token的作用就来了，我们上面在签发token的时候已经将用户名放在了token里面，每次发请求前端都会把token放在请求头带过来，因此后端可以在请求头直接取出来，然后解密获得用户名，再根据用户名从数据库获取密码，如果能用密码解密，说明这个token是合法的，即请求的用户认证成功，然后就可以执行下面的业务了。

当然啦，由于这部分逻辑对于需要验证用户身份的业务来说，每次执行的流程都是一样的，因此我们才把它封装在拦截器里面，在请求到达controller之前拦截下来，鉴权通过后再放行，不通过则抛出JWT异常，由全局异常处理器捕获，然后返回错误信息，那么全局异常处理器是什么，为什么要用全局异常，处理器呢？

# 3、添加自定义注解

现在还有一个问题，那就是有的方法是不需要鉴权的，如何知道当前的方法需不需要鉴权？

其实在配置拦截器的时候我们会配置它的拦截路径，我们根据方法的请求路径使用 addPathPatterns("路径") 方法配置，如果方法需要鉴权，就添加一个这个方法，但是随着方法数量的增多，这显然不是一种优雅的方案，所以应该怎么解决呢？

我们就直接一不做二不休拦截所有路径不就行了？

不行，因为刚刚说了，有的路径是不需要拦截的。那咋办？

我们可以这样玩，先拦截所有路径，然后采用注解的方式，在需要鉴权的方法上添加自定义注解，比如名字可以叫 @RequireToken，然后在需要鉴权的方法上添加另一个自定义注解，比如名字可以叫 @PassToken。在Jwt拦截器进行鉴权之前先判断请求的方法上是否有跳过鉴权的注解 @RequireToken ，有的话则放行，没有的话继续判断是否有 @RequireToken 注解，有的话则鉴权，没有的话再放行，不过在这个项目里面，我们不需要鉴权的方法都要加 @PassToken 注解，需要鉴权的方法都要加上 @RequireToken 注解

接下来在 demo 下新建包 annotation，包下新建两个注解@RequireToken 和 @PassToken：

# 4、Jwt拦截器

在 demo 下新建包 interceptor，包下新建 JwtInterceptor 类：

package com.example.demo.interceptor;

import com.auth0.jwt.JWT;
import com.auth0.jwt.JWTVerifier;
import com.auth0.jwt.algorithms.Algorithm;
import com.auth0.jwt.exceptions.JWTDecodeException;
import com.auth0.jwt.exceptions.SignatureVerificationException;
import com.auth0.jwt.exceptions.TokenExpiredException;
import com.example.demo.annotation.PassToken;
import com.example.demo.annotation.RequireToken;
import com.example.demo.entity.User;
import com.example.demo.exception.JwtException;
import com.example.demo.message.Message;
import com.example.demo.service.UserService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;
import org.springframework.web.method.HandlerMethod;
import org.springframework.web.servlet.HandlerInterceptor;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.lang.reflect.Method;

/**
 * Jwt拦截器
 */
@Component
public class JwtInterceptor implements HandlerInterceptor {
    @Autowired
    private UserService userService;

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) {
        String token = request.getHeader("token");
        // 如果不是映射到方法直接通过
        if (!(handler instanceof HandlerMethod)) {
            return true;
        }
        Method method = ((HandlerMethod) handler).getMethod();
        //检查是否有PassToken注解，有则跳过认证
        if (method.isAnnotationPresent(PassToken.class)) {
            PassToken passToken = method.getAnnotation(PassToken.class);
            if (passToken.required()) {
                return true;
            }
        }
        //检查有没有需要用户权限的注解
        if (method.isAnnotationPresent(RequireToken.class)) {
            RequireToken requireToken = method.getAnnotation(RequireToken.class);
            if (requireToken.required()) {
                // 执行认证
                if (token == null) {
                    throw new JwtException(Message.NO_AUTHORITY, "无token，请重新登录");
                }
                // 获取 token 中的 username
                String username;
                try {
                    username = JWT.decode(token).getAudience().get(0);
                } catch (JWTDecodeException j) {
                    throw new JwtException(Message.NO_AUTHORITY, "token不合法");
                }
                User user = userService.findUserByUserName(username);
                if (user == null) {
                    throw new JwtException(Message.NO_AUTHORITY, "用户不存在，请重新登录");
                }
                // 验证 token
                JWTVerifier jwtVerifier = JWT.require(Algorithm.HMAC256(user.getPassword())).build();
                try {
                    jwtVerifier.verify(token);
                } catch (SignatureVerificationException e) {
                    throw new JwtException(Message.NO_AUTHORITY, "无效签名!");
                } catch (TokenExpiredException e) {
                    throw new JwtException(Message.NO_AUTHORITY, "token过期!");
                }
                return true;
            }
        }
        return false;
    }
}

package com.example.demo.resolver;

import com.auth0.jwt.JWT;

import org.springframework.beans.factory.annotation.Autowired;

import org.springframework.stereotype.Component;
import org.springframework.web.bind.support.WebDataBinderFactory;
import org.springframework.web.context.request.NativeWebRequest;
import org.springframework.web.method.support.HandlerMethodArgumentResolver;
import org.springframework.web.method.support.ModelAndViewContainer;

/**
 * 实现注入用户信息到使用了@userInfo注解的方法参数里
 */
@Component
public class UserInfoHandlerMethodArgumentResolver implements HandlerMethodArgumentResolver {
    @Autowired
    private UserService userService;

    @Override
    public boolean supportsParameter(MethodParameter parameter) {
        return parameter.hasParameterAnnotation(UserInfo.class) &&
                parameter.getParameterType().isAssignableFrom(User.class);
    }

    @Override
    public Object resolveArgument(MethodParameter parameter, ModelAndViewContainer container,
                                  NativeWebRequest request, WebDataBinderFactory factory) {
        // header中获取用户token
        String token = request.getHeader("token");
        if (token != null) {
            String username = JWT.decode(token).getAudience().get(0);
            return userService.findUserByUserName(username); // 返回用户信息
        }
        return false;
    }
}

在exception 包下新建 GlobalExceptionHandler 全局异常处理类，除了Jwt异常，在这里我们可以对一些常见的异常进行输出，比如方法请求类型异常，有时候我们Get搞成了Post，就会抛出这个异常，我们可以返回错误信息方便调试

package com.example.demo.exception;

import com.example.demo.message.Message;
import com.example.demo.vo.Result;
import org.springframework.web.HttpRequestMethodNotSupportedException;
import org.springframework.web.bind.annotation.ControllerAdvice;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.ResponseBody;

/**
 * 全局异常处理器
 */
@ControllerAdvice
public class GlobalExceptionHandler {
    @ResponseBody
    @ExceptionHandler(Exception.class)
    public Object handleException(Exception e) {
        e.printStackTrace();
        // Jwt异常
        if (e instanceof JwtException) {
            return new Result(((JwtException) e).getCode(), e.getMessage(), null);
        }
        // 请求方法类型异常
        if (e instanceof HttpRequestMethodNotSupportedException) {
            return new Result(500, e.getMessage(), null);
        }
        // 其他异常统一返回：“服务器繁忙”
        return new Result(Message.ERROR, Message.ERROR_MESSAGE, null);
    }
}

# 8、完善

在该加上注解的地方加上注解

package com.example.demo.api;


import com.auth0.jwt.JWT;
import com.auth0.jwt.algorithms.Algorithm;
import com.example.demo.annotation.PassToken;
import com.example.demo.annotation.RequireToken;
import com.example.demo.annotation.UserInfo;
import com.example.demo.entity.User;
import com.example.demo.message.UserMessage;
import com.example.demo.service.UserService;
import com.example.demo.message.Message;
import com.example.demo.vo.Result;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

import java.util.Calendar;

@RestController
@RequestMapping("/user")
public class UserApi {
  @Autowired
  private UserService userService;

  /**
  	* 获取所有用户列表（仅用于测试）
  	*
  	* @return 所有用户列表
  	*/
  @GetMapping("/list")
  @PassToken
  public Result getUserList() {
    return new Result(Message.SUCCESS, Message.SUCCESS_MESSAGE, userService.selectAll());
  }

  /**
  	* 登录
  	*
  	* @param user 用户信息
  	* @return token
  	*/
  @PostMapping("/login")
  @PassToken
  public Result login(@RequestBody User user) {
    if (user.getUsername() == null || user.getUsername().length() == 0
        || user.getPassword() == null || user.getPassword().length() == 0) {
      return new Result(Message.PARAM_ERROR, UserMessage.ERROR_USERNAME_OR_PASSWORD, Result.NO_DATA);
    }
    User sqlUser = userService.findUserByUserName(user.getUsername());
    if (sqlUser != null) {
      if (user.getPassword().equals(sqlUser.getPassword())) {
        String token = generateToken(user);
        return new Result(Message.SUCCESS, UserMessage.SUCCESS_LOGIN, token);
      } else {
        return new Result(Message.PARAM_ERROR, UserMessage.ERROR_USERNAME_OR_PASSWORD, Result.NO_DATA);
      }
    } else {
      return new Result(Message.PARAM_ERROR, UserMessage.ERROR_USERNAME_OR_PASSWORD, Result.NO_DATA);
    }
  }

  /**
  	* 签发token，并且设置有效期
  	*
  	* @param user 用户信息
  	* @return token
  	*/
  public String generateToken(User user) {
    Calendar instance = Calendar.getInstance();
    instance.add(Calendar.SECOND, 3600 * 24 * 7);
    return JWT.create()
      .withAudience(user.getUsername())
      .withExpiresAt(instance.getTime())
      .sign(Algorithm.HMAC256(user.getPassword())); // 使用用户密码作为私钥
  }

  /**
  	* 获取用户信息
  	*
  	* @param userInfo 用户信息
  	* @return 用户信息
  	*/
  @GetMapping("/getUserInfo")
  @RequireToken
  public Result getUserInfo(@UserInfo User userInfo) {
    userInfo.setId(0); // 设置id为0，防止泄露
    userInfo.setPassword(""); // 设置密码为空字符串，防止密码泄露
    return new Result(Message.SUCCESS, Message.SUCCESS_MESSAGE, userInfo);
  }
}
